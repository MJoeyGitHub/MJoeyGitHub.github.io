swift Class和Struct

发表于 2016-05-30 | 分类于 Swift

谈谈什么是Class，Struct。 以及两者直接的差异。以及适用环境

Class

引用类型

通过在 class后接类名称来创建一个类。在类里边声明属性与声明常量或者变量的方法是相同的，唯一的区别的它们在类环境下。同样的，方法和函数的声明也是相同的写法。

通过在类名字后边添加一对圆括号来创建一个类的实例。使用点语法来访问实例里的属性和方法

getter 和 setter

willSet和 didSet –
如果你不需要计算属性但仍然需要在设置一个新值的前后执行代码

Class 和 Struct的区别

Class表达一个具体的对象

Struct表达一个普通的值

class和Struct对于常量的理解是不同的 应用上面的例子

我们可以理解为用于p1是一个值类型，当然他的值不希望被改动
而p2是对象类型 常量对他的含义是 他不能指向其他的对象

class和Struct对于比较而言也有异同

Struct中的方法 默认是不能修改内部的Member（成员变量）只能在内部方法的前面加上关键字 mutating 。 然而Class却不用添加

class 是可以继承,也可以被继承 struct不能被继承

struct和class都可以遵守protocol协议

那么到底什么时候用Class 什么时候用Struct呢？

Class 是引用语义，是动态的操作堆里面的数据，只是复制引用对象，并没有复制真正的值

Struct 是值语义， 是直接操做栈里面的源数据，是复制了一块新的值

Class更重量级， 他能完成Struct几乎所有的操作，Struct更轻量级

用小东西 完成大事情！！

tps： 想清楚我们究竟想表达一个对象，还是一个引用的值 就可以灵活的运用在项目上了
