git指令上传项目到远程仓库

发表于 2017-04-26 | 分类于 git

如何上传代码到gitHub？ 本文有详细的使用介绍

利用git上传本地项目到gitHub上面

先提交到本地仓库

首先本地创建一段代码

登录你的gitHub的账号

打开你的终端， cd 你的代码在电脑所在的位置

1.初始化本地仓库

$ git init

2.添加文件到暂缓区

$ git add .

这里友情提示一下，如何知道自己的文件是否上传到暂缓区了呢？

$ git status // 如果有绿色的文字在终端显示， 说明上传成功

3.从暂缓区提交到本地仓库

$ git commit -m ‘听说这个是注释？’

当然可以去掉 -m 后面的内容

4.打本地标签（给当前版本的代码做一个版本控制）

$ git tag -a ‘输入你的版本号（按照标准标签格式即可）’ -m ‘打标签标注标签功能’

5.查看本地标签时候录用成功

$ git tag

以上才做仅仅是提交到本地仓库里面

提交到gitHub仓库

  1. 关联到远程仓库

    $ git remote add origin 你的远程仓库的地址

  2. 上传代码到远程仓库

    $ git push origin master // 注意上步操作之后，如果是第一次创建仓库，需要填写账户的一下信息 已经密码 依次填写就好

  3. 如有标签上传 许上传本地标签

    $ git push –tags

如果你的项目的代码有更新了~ 需要重新上传，以及递增你的新标签

首先查看你项目的代码是否真的被更改过(可以忽略)

$ git status // 如果你的终端上 出现了红色的字体， 说明有更新

更新代码到你的本地暂缓区

$ git add .

提交代码到本地仓库

$ git commit -m ‘注释，可以不写’

打标签

$ git tag ‘0.0.2’

上传标签定义的代码

$ git push –tags

如果你觉得你之前的上传的代码有严重的BUG 或者 其他原因想要删除你之前的标签

git push origin :标签 // 注意 是冒号
