开发框架到cocoapods里面给他人使用

搭建好本地的环境

  1. 首先把自己写好的代码上传到原远程仓库内
    如何上传？
  1. 创建一个关于这个仓库的描述文件 格式为 spec

$ cd 到你项目的文件夹
$ pod spec create 你的项目名称

如上步操作完之后， 你会发现在你的项目文件内多了一个spec文件， 右键用Xcode 打开


  s.name         = "MyLib"                          -> 你的框架的名称
  s.version      = "0.0.1"                          ->  版本号
  s.summary      = "A short description of MyLib."  ->  这个是你的项目的简介
   s.description  = 这个是这个项目的详细描述 ， 注意一定要比summary字数多才可以正常使用


  s.homepage     = "http://EXAMPLE/MyLib"  -> 这里添加你项目仓库的地址 注意后面是没有 .git的地址

 s.source       = { :git => "http://EXAMPLE/MyLib.git", :tag => "#{s.version}" }  ->  这里要填写你的仓库的地址  注意是后面带.git的     tag是填写你打标签的版本号

s.source_files  = "Classes", "Classes/**/*.{h,m}"  ->   这个是让别人pod的你代码的时候只pod 能用到项目上的文件， 去掉没用的代码
s.exclude_files = "Classes/Exclude"  ->   这个一般不会用 用 #注释  注释就行了

上传你设置后的spec

设置Trunk 这是官方文档

$ pod trunk register 你的邮箱（能用的，因为要发送验证邮件） ‘你的用户名’ –verbose

提示： ACE， YOU’RE SET UP!!!!!1

$ pod trunk push 你的spec名字

如果操作成功， 就会进入审核阶段！ 审核成功之后， 就可以用cocoapods 来使用你的代码啦~
