可观察序列(Observable)和观察者(Observer）

别TM乱动，交警瞅着呢！

咱们就是可观察的序列Observable，交警就是观察者Observer

按钮的点击事件！

当点击App内部的按钮的时候，我们把按钮的每一次点击看做成可观察的序列， RxSwift通过subscribe来订阅这个点击事件，每一次点击都会发送一个数据，订阅者的subscribe收到这个数据进行相应的响应

可观察序列(Observable)

序列有三种情况

查找下一条数据(Next)

序列中遇到问题(Error)

整条序列执行完毕(Completed)。

观察者 (Observer)

观察者中有一个订阅的功能

两者在开发阶段之间的关系

序列Observable做出“动作”

观察者Observer订阅序列，然后收到序列的值，进行操作处理，直到完成

RxSwift的基本操作符，以及含义

创建

  • create 创建序列
  • just 只创建包含一个元素的序列。换言之，只发送一个值和 .Completed
  • deferred 只有在有观察者订阅时，才去创建序列
  • empty 创建一个空的序列，只发射一个 .Completed
  • error 创建一个发射 error 终止的序列
  • toObservable 使用 SequenceType 创建序列
  • interval 创建一个每隔一段时间就发射的递增序列
  • of 通过一组元素创建一个序列
  • range 创建一个有范围的递增序列
  • repeatElement 创建一个发射重复值的序列
  • timer 创建一个带延迟的序列

其他

  • never 不创建序列，也不发送通知
  • asObservable 返回一个序列
