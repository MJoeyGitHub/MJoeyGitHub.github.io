UICollectionView自定义布局详解

发表于 2017-06-06 | 分类于 iOS技术

关于一些电商的首页，经常会看到一些，看似毫无规律的布局，介绍一个那些电商绝大多数的布局情况，当然不排除一切其他特殊的情况.本文以自定义UICollectionView布局来讲

两者之间的差别

UICollectionViewFlowLayout（经常用的流水布局，以下简称FlowLayout）是继承自UICollectionViewLayout（以下简称Layout）的.

注意，Layout除了帮我们处理了重用的功能之外基本上什么都没有，里面没有FlowLayout的一些属性

这也是他的强大之处，给我们最大的想象空间去创作，当然没我说的那么夸张，无非就市面上的那几种布局 （令人不禁唏嘘，我们到底都干了些什么？）

自定义布局，从哪里开始入手？

首先我们分析，如果是自定义布局， 我们到底缺失了什么？

  • 每个Item的位置，啥时候才被初始化？
  • 能滚动，ContentSize
  • 布局的一写辅助属性

首先能想到的就这么多。 那么开就就上面的以为逐一攻破！看看苹果给了我们什么。

辅助属性

这个方法是返回某个特定区域的布局的属性的数据集合。
这个数组中存放的都是UICollectionViewLayoutAttributes对象。
UICollectionViewLayoutAttributes对象决定了cell的排布方式（frame等）。

如果在这个方法外部设置某些属性的时候，需要调用layoutAttributesForItemAtIndexPath方法

一旦重新刷新布局，就会按顺序调用下面的方法：

参数：

proposedContentOffset：默认情况下CollectionView停止滚动时最终的偏移量。

velocity：滚动速率，通过这个参数可以了解滚动的方向。
说明：返回值决定了collectionView停止滚动时最终的偏移量contentOffset

注意：拖动会有初速度和加速度，这两个参数是不可见的，系统会根据这两个速度算出一个向量速度也就是velocity提供给我们来判定速度方向。当然如果是拖动手势而不是拖甩手势也就是没有加速度，velocity直观的反应就是0。

这个方法，一般是做一些比较炫酷的东西，以炫酷的轮播图为例， 当手动滑动的时候，可以设置每一个Item的悬停位置。
